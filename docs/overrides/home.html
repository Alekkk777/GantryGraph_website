agent.py 
from gantrygraph import GantryEngine, gantry_tool
from gantrygraph.perception import DesktopScreen
from gantrygraph.actions import MouseKeyboardTools
from langchain_anthropic import ChatAnthropic

@gantry_tool
async def send_slack(channel: str, msg: str) -> str:
    """Post a message to Slack."""
    return await slack.post(channel, msg)

agent = GantryEngine(
    llm=ChatAnthropic(model="claude-sonnet-4-6"),
    perception=DesktopScreen(),
    tools=[MouseKeyboardTools(), send_slack],
    max_steps=30,
)
agent.run("File the quarterly report and ping #finance")

What can your agent do?

GantryGraph gives your LLM eyes, hands, and guardrails.

Control the desktop

The agent takes a screenshot, decides what to click or type, and acts — just like a human at a keyboard. Works on any GUI application without integrations.

Desktop guide →

Browse the web

Navigate URLs, fill forms, extract data, and interact with web apps — driven by Playwright with full visual context at every step.

Browser guide →

Call any tool or API

Wrap any Python function with @gantry_tool and it's instantly available to the LLM. Connect GitHub, Notion, Postgres, or any service via the MCP standard.

MCP guide →

Stay in control

Set a workspace sandbox so the agent can't touch files outside your project. Require human approval before destructive actions. Hard-cap cost and runtime with budget limits.

Safety guide →

Simple loop. No magic.

Four deterministic steps. Every decision is observable and logged.

01
Observe

Take a screenshot, render the DOM, or read any custom sensor. The agent always knows what it's looking at.

→
02
Think

The LLM receives the observation and decides what to do next — or declares the task done.

→
03
Act

Tool calls run through the guardrail gate. Errors are fed back automatically for self-correction.

→
04
Review

No more tool calls? Task complete. Otherwise loop back — up to max_steps.
